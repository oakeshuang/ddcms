Hexo的基本使用

Hexo 官网

1. 本地安装

npm install -g hexo

2. 本地创建博客

初始化

hexo init

生成的文件：

node_modules：是依赖包
public：存放的是生成的页面
scaffolds：命令生成文章等的模板
source：用命令创建的各种文章
themes：主题
_config.yml：整个博客的配置
db.json：source解析所得到的
package.json：项目所需模块项目的配置信息

3. github上搭建好自己的 博客，如 yourname.github.io

如何搭建自己的github博客

4. 配置

修改_config.yml文件的一些配置

注意：冒号之后都是有一个半角空格的

deploy:
type: git
repo: https://github.com/YourgithubName/YourgithubName.github.io.git

5. 安装本地预览包以及博客上传包

安装hexo本地服务器,用于本地预览

npm i hexo-server

推送博客到github

npm install hexo-deployer-git --save

6. 创建一篇文章 More

hexo new 文章名

7. 本地预览 More

hexo server

8. 生成静态文件 More

hexo generate

9. 推送到github More

hexo deploy

10. 一般操作

hexo clean // 清除原来生成的文件
hexo generate // 重新生成文件
hexo server    // 本地预览

hexo clean // 清除原来生成的文件
hexo generate // 重新生成文件
hexo deploy    // 推送到github

1. 修改语言包

配置文件在_config.yml ，一般的网站设置都在这里改，比如：改语言包，网站标题，作者等

2. 生成自定义模板文件

可以在scaffokds里面自定义一个自己的模板文件，然后执行 hexo new 模板文件名 生成的文件名，

会按照模板文件里面的配置生成一个模板到生成的文件里面
